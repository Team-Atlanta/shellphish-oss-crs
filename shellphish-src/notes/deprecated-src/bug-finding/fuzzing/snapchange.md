# Snapchange

Snapchange is a snapshot-based fuzzer using KVM hypervisor that the CRS uses for Linux kernel fuzzing. It captures VM state at target entry points and fuzzes from the snapshot for fast iterations without full system boot.

## Purpose

- Snapshot-based kernel fuzzing
- Fast reset via VM snapshots (no full boot)
- System call sequence fuzzing
- Grammar-guided syscall generation

## CRS-Specific Usage

**Pipeline**: [`pipeline.yaml`](https://github.com/sslab-gatech/shellphish-afc-crs/blob/main/components/snapchange/pipeline.yaml)

### Workflow

**1. Build Target** - `snapchange_build` ([Lines 28-69](https://github.com/sslab-gatech/shellphish-afc-crs/blob/main/components/snapchange/pipeline.yaml#L28-L69)):
- Builds Linux kernel or target binary
- Prepares for snapshotting

**2. Take Snapshot** - `snapchange_take_snapshot` ([Lines 71-129](https://github.com/sslab-gatech/shellphish-afc-crs/blob/main/components/snapchange/pipeline.yaml#L71-L129)):
- Uses QEMU to boot VM
- Captures state at harness entry point
- Saves snapshot for fuzzing

**3. Fuzz from Snapshot** - `snapchange_fuzz` ([Lines 131-230](https://github.com/sslab-gatech/shellphish-afc-crs/blob/main/components/snapchange/pipeline.yaml#L131-L230)):
- Restores snapshot for each iteration
- Grammar-guided syscall sequences
- Tracks coverage per input

### Grammar Integration

**Syzlang Grammar** ([Lines 163-165](https://github.com/sslab-gatech/shellphish-afc-crs/blob/main/components/snapchange/pipeline.yaml#L163-L165)):

- System call sequence specifications

- Generated by syzgrammar-gen component

### Implementation

**Language**: Rust ([`Cargo.toml`](https://github.com/sslab-gatech/shellphish-afc-crs/blob/main/components/snapchange/snapchange/Cargo.toml))

**Key Features**:
- KVM-based snapshot restore
- Coverage tracking via KVM breakpoints
- QEMU integration for snapshot creation
